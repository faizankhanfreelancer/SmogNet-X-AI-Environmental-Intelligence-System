## Project Overview

SmogNet is a full-stack air quality intelligence system that:

- **Detects** pollution spikes using Rolling Baseline + Isolation Forest (ML)
- **Classifies** probable pollution sources (Crop Burning, Vehicular, Industrial, Dust Storm, Mixed)
- **Generates** plain-language health alerts in English and Urdu
- **Visualises** real-time data on an interactive React dashboard + Streamlit analytics
- **Predicts** future AQI levels using seasonal + diurnal forecasting models

**8 Pakistani cities covered:** Lahore, Karachi, Islamabad, Peshawar, Multan, Faisalabad, Quetta, Rawalpindi

**11,680 synthetic readings** seeded into PostgreSQL with realistic seasonal patterns.

## Quick Start

### Prerequisites
- Node.js 20+ and pnpm 9+
- Python 3.9+
- PostgreSQL 14+

### 1. Clone and install dependencies

```bash
git clone <repo-url>
cd SmogNet-AI-Air-Quality-System
pnpm install
```

### 2. Set environment variables

```bash
# Copy and fill in your Postgres connection string
export DATABASE_URL="postgresql://user:password@localhost:5432/smognet"
export SESSION_SECRET="your-secret-key-here"
```

### 3. Set up the database

```bash
# Push schema to Postgres
pnpm --filter @workspace/db run push

# Seed with 11,680 synthetic readings
pnpm --filter @workspace/db run seed
```

### 4. Start the API server

```bash
pnpm --filter @workspace/api-server run dev
# Server starts on port 8080, accessible at /api
```

### 5. Start the React dashboard

```bash
pnpm --filter @workspace/smognet run dev
# Dashboard starts on port 5173 (or $PORT)
```

## Python ML Pipeline

### Setup

```bash
cd smognet
pip install -r requirements.txt
```

### Run the full AI pipeline

```bash
python run_pipeline.py
```

This runs:
1. **Data Loader** — loads/generates air quality readings for 8 cities
2. **Preprocessor** — cleans, normalises, feature-engineers the data
3. **Anomaly Detector** — rolling baseline + Isolation Forest spike detection
4. **Source Classifier** — rule-based + Random Forest source attribution
5. **Alert Generator** — generates English + Urdu health alerts
6. Saves outputs to `smognet/outputs/`

### Run the Streamlit dashboard

```bash
# Make sure pipeline has been run first (generates processed data)
python run_dashboard.py
# Dashboard opens at http://localhost:8501
```

**Streamlit pages:**
- 📊 **Overview** — national KPIs, city stats
- 🌍 **Interactive Map** — GIS choropleth with 3 visualisation modes
- 🔬 **Analytics** — deep-dive charts, seasonal analysis
- ⚡ **Anomaly Detection** — spike timeline, detector comparison
- 🏭 **Source Classification** — source attribution results
- 🚨 **Alert Center** — generated health alerts
- 🤖 **Live Prediction** — real-time AI forecasting

## Dataset

**11,680 synthetic air quality readings** across 8 Pakistani cities:

| City | Province | Key Pollution Source |
|------|----------|----------------------|
| Lahore | Punjab | Crop Burning |
| Karachi | Sindh | Vehicular Emissions |
| Islamabad | Federal | Mixed Sources |
| Peshawar | KPK | Crop Burning |
| Multan | Punjab | Dust Storm |
| Faisalabad | Punjab | Industrial Emissions |
| Quetta | Balochistan | Dust Storm |
| Rawalpindi | Punjab | Vehicular Emissions |

**Seasonal patterns:** Winter (Nov–Feb) = highest pollution (crop burning + temperature inversions). Monsoon (Jul–Sep) = lowest pollution. Rush hours (07–09, 17–20) show elevated readings.

**Pollutants tracked:** PM2.5, PM10, AQI, CO, O₃, NOₓ, Temperature, Humidity.
